Hexo博客+Next+GithubPage自定义域名支持https

发表于 2019-07-24 更新于 2019-07-25
本文字数： 589 阅读时长 ≈ 1 分钟

搭建了博客之后，Chrome打开博客时没有小绿锁感觉有点别扭，于是想把域名支持https。网上搜了之后，发现GithubPage支持https还挺容易的，但是也遇到了一些问题。

参考：使用 HTTPS 保护 GitHub Pages 站点

根据域名解析类型分为两类

  • CNAME解析

这种类型需要在你的Github Pages所在的仓库中设置开启Enforce HTTPS即可。

如果发现复选框为黑色开启不了，可以将custom domian那一栏的内容先删除为空，再点击save保存，接着再次填入你的自定义域名，点击save保存。这时会出现下面的提示：


这表示你的证书还在准备中，还没发完，一般等待一两个小时就可以了。

  • A解析

如果你的解析为A记录的话，将解析的IP换为以下四个之一即可：

我所遇到的问题

经过上面的步骤，一般的站点应该等待几小时之后就没问题了，但是我的博客是用Hexo加Next主题搭建在Github Pages上面的，就遇到了一点问题，其实这个问题在官方指导页面中也提到了，官方界面。

其实主要就是next主题在前端加载某些资源的时候，用的是HTTP协议，只要把这些改成HTTPS协议就好。
