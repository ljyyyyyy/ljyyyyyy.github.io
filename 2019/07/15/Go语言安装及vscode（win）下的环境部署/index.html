Go语言安装及vscode（win）下的环境部署

发表于 2019-07-15 更新于 2019-07-18
本文字数： 1.1k 阅读时长 ≈ 1 分钟

Go语言现在并没有成熟的IDE，所以我照着网上的教程用我很喜欢的VS Code搭建了一个调试环境。

前期准备工作

电脑上已经安装好Git工具及VS Code，另外我在极少一些博客中看到还需要安装Node.js,但其他博客中并未看到需要安装Node.js，不清楚具体是否需要安装Node.js。

安装Golang

这个不用多说，直接去官网选择自己需要的版本下载安装即可。由于Wall的存在，go的官网并不总是能访问，但go在国内有一个官方的镜像。

Golang官网
国内镜像

配置环境变量

安装好之后按打开命令行，输入运行go env可以查看golang的环境配置信息和go的环境变量。其中，最重要的有两个：GOROOT和GOPATH。

  • GOROOT：该变量指向go的安装目录，默认一般为C:\go。
  • GOPATH：该变量为go的工程目录，不能是go的安装目录。它可以有多个目录，目录之间用;来分隔，且下面安装的Go Tools会安装在第一个目录。

注意：
在安装Go Tools这一步可能会失败多次。第一步中的git clone如果用https的方式很可能失败，错误提示为：
error: RPC failed; curl 56 OpenSSL SSL read:SSL ERROR SYSCALL, errno 10054。
网上找了一下解决方法主要有两种：一说是安全设置的原因，这种解决方法是在设置中关闭github的SSL验证
git config http.sslVerify "false";
另一种说法是http或是https协议设置缓存太小的原因，这种解决方法是在设置中将推送的缓存扩大
git config http.postBuffer 524288000。
但这两种方法似乎都没对我起到太大的作用，我觉得主要是有些下载被Wall阻止了……
